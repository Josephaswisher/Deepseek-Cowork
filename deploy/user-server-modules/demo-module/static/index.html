功能概述

自定义模块功能允许用户扩展 deepseek-cowork 服务端能力，无需修改核心代码。主要特点：

  • 独立的模块目录，便于管理和版本控制
  • 标准化的模块接口，简化开发流程
  • 支持路由注册、静态资源、事件发射
  • 热加载支持，服务重启即可生效

基础 API 接口

  • GET /demo/ 本介绍页面
  • GET /api/demo/status 获取模块运行状态
  • POST /api/demo/echo Echo 请求体
  • GET /api/demo/info 获取模块详细信息

模块接口规范

自定义模块需要导出一个 setup 函数，返回实现以下接口的服务实例：

方法 必须 说明
init() 可选 初始化逻辑，在模块加载后调用
setupRoutes(app) 可选 注册 Express 路由
start() 可选 启动服务
stop() 可选 停止服务，清理资源

快速开发指南

创建模块目录
# Windows %APPDATA%\deepseek-cowork\user-server-modules\my-module\ # macOS ~/Library/Application Support/deepseek-cowork/user-server-modules/my-module/ # Linux ~/.config/deepseek-cowork/user-server-modules/my-module/
编写模块入口 (index.js) - 带核心服务集成
const { EventEmitter } = require('events'); function setupMyModuleService(options = {}) { // 从 options 获取注入的核心服务 const HappyService = options.HappyService; const MessageStore = options.MessageStore; class MyModuleService extends EventEmitter { constructor() { super(); this.happyService = HappyService; this.messageStore = MessageStore; } async init() { console.log('[MyModule] 初始化'); // 监听 AI 消息事件 if (this.happyService) { this.happyService.on('happy:message', (msg) => { console.log('收到消息:', msg.role); }); } } setupRoutes(app) { // 调用 HappyService 发送消息 app.post('/api/my-module/send', async (req, res) => { if (this.happyService) { await this.happyService.sendMessage(req.body.message); res.json({ success: true }); } }); // 调用 MessageStore 获取历史 app.get('/api/my-module/history', (req, res) => { if (this.messageStore) { const msgs = this.messageStore.getMessages(req.query.sessionId); res.json({ messages: msgs }); } }); } async start() { this.emit('started'); } async stop() { this.emit('stopped'); } } return new MyModuleService(); } module.exports = { setupMyModuleService };
创建配置文件 (userServerModulesConfig.js) - 注入核心服务
module.exports = { modules: [ { name: 'my-module', module: './my-module', setupFunction: 'setupMyModuleService', enabled: true, features: { hasRoutes: true }, // 通过 getOptions 注入核心服务 getOptions: (config, runtimeContext) => ({ HappyService: runtimeContext?.services?.HappyService, MessageStore: runtimeContext?.services?.MessageStore }) } ] };
重启服务

服务重启后会自动加载新模块，访问 /api/my-module/hello 验证。

部署命令

# 查看可用模块模板 node deploy/index.js module --list # 部署演示模块 node deploy/index.js module demo-module # 查看已部署模块状态 node deploy/index.js module --status # 移除已部署模块 node deploy/index.js module --remove demo-module
